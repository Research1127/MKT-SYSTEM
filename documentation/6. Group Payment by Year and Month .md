# Group Payment By Year and Month

## Change Query Handler

Fullcode:
```csharp
public class StudentPaymentQueryHandler(
    ILogger<StudentPaymentQueryHandler> logger,
    IStudentRepository studentRepository
) : IRequestHandler<StudentPaymentQuery, StudentPaymentResponseDto>
{
    public async Task<StudentPaymentResponseDto> Handle(StudentPaymentQuery request, CancellationToken cancellationToken)
    {
        logger.LogInformation("Retrieving payments for IC: {IcNumber}", request.IcNumber);

        var student = await studentRepository.GetStudentWithPaymentsByIc(request.IcNumber, cancellationToken);

        if (student == null)
        {
            logger.LogWarning("Student not found: {IcNumber}", request.IcNumber);
            return null;
        }

        var monthlyFee = student.Class.MonthlyFee;

        decimal totalDue = 0;
        decimal totalOutstanding = 0;
        decimal totalPaid = 0;

        // Group payments by Month + Year
        var groupedPayments = student.Payments
            .GroupBy(p => new { p.Month, p.Year })
            .Select(g =>
            {
                var totalPaidInMonth = g.Sum(p => p.PaidAmount);
                var dueAmount = monthlyFee;
                var outstandingAmount = dueAmount - totalPaidInMonth;

                // Monthly Status
                string monthlyStatus;
                if (totalPaidInMonth == 0)
                    monthlyStatus = "Unpaid";
                else if (totalPaidInMonth < dueAmount)
                    monthlyStatus = "Partially";
                else
                    monthlyStatus = "Paid";

                // Increment totals
                totalDue += dueAmount;
                totalPaid += totalPaidInMonth;
                totalOutstanding += outstandingAmount;

                return new PaymentDto
                {
                    Month = g.Key.Month.ToString(),
                    Year = g.Key.Year,
                    PaidAmount = totalPaidInMonth,
                    DueAmount = dueAmount,
                    OutstandingAmount = outstandingAmount,
                    PaymentStatus = monthlyStatus
                };
            })
            .OrderBy(p => p.Year)
            .ThenBy(p => p.Month)
            .ToList();

        // Overall Status
        string overallStatus;
        if (totalPaid == 0)
            overallStatus = "No fees have been paid yet";
        else if (totalPaid < totalDue)
            overallStatus = "Partial fees paid, some outstanding";
        else
            overallStatus = "All fees are paid";

        // Build response
        return new StudentPaymentResponseDto
        {
            Name = student.Name,
            IcNumber = student.IcNumber,
            Payments = groupedPayments,
            TotalPaid = totalPaid,
            TotalDue = totalDue,
            TotalOutstanding = totalOutstanding,
            OverallStatus = overallStatus
        };
    }
}
```

---

# Explanation

## Step 1: Group payments by month/year

We need to sum all payments per month before calculating status.

In C# with LINQ:

```csharp
var groupedPayments = student.Payments
    .GroupBy(p => new { p.Month, p.Year });
```

- GroupBy creates a group for each unique combination of Month and Year.

- So if there are multiple payments in Month 2, Year 2026, they will all be in one group.

---

## Step 2: Sum total paid per month

For each group:

```csharp
var totalPaidInMonth = g.Sum(p => p.PaidAmount);
```

- g is the group of payments for the same month.
- Sum() adds all amounts in that group.
- This gives the true total paid for the month.

---

## Step 3: Calculate monthly due, outstanding, and status

Now we know the monthly fee (from the class) and total paid in month:

```csharp
var dueAmount = monthlyFee;
var outstandingAmount = dueAmount - totalPaidInMonth;

string monthlyStatus;
if (totalPaidInMonth == 0)
    monthlyStatus = "Unpaid";
else if (totalPaidInMonth < dueAmount)
    monthlyStatus = "Partially";
else
    monthlyStatus = "Paid";
```

- outstandingAmount is now accurate, even if multiple payments exist.
- Status is based on sum of payments, not per transaction.

---

## Step 4: Increment overall totals

We also need total paid, total due, and total outstanding for the student:

```csharp
totalDue += dueAmount;
totalPaid += totalPaidInMonth;
totalOutstanding += outstandingAmount;
```
- Do this inside the grouped loop so totals are correct.

---

## Step 5: Calculate overall status

After looping through all months:

```csharp
string overallStatus;
if (totalPaid == 0)
    overallStatus = "No fees have been paid yet";
else if (totalPaid < totalDue)
    overallStatus = "Partial fees paid, some outstanding";
else
    overallStatus = "All fees are paid";
```

- This gives a sentence summary for the student, based on all months.

---

## Step 6: Build the DTO for frontend

```csharp
paymentDtos.Add(new PaymentDto
{
    Month = g.Key.Month.ToString(),
    Year = g.Key.Year,
    PaidAmount = totalPaidInMonth,
    DueAmount = dueAmount,
    OutstandingAmount = outstandingAmount,
    PaymentStatus = monthlyStatus
});
```

- One row per month, no duplicate “Partially” rows.
- Frontend sees clean data.

### Field-by-field explanation

#### Month = g.Key.Month.ToString()
- Retrieves the month value from the group key.
- g.Key represents the grouping criteria (Month and Year).
- Converted to string for display purposes in the UI.

#### Year = g.Key.Year
- Retrieves the year value from the group key.
- Ensures the payment is correctly associated with the right year.

#### PaidAmount = totalPaidInMonth
- Represents the total amount paid for that specific month.
- Calculated by summing all payment transactions within the group.

#### DueAmount = dueAmount
- Represents the expected monthly fee for the student.
- This value is typically derived from the student’s class configuration.

#### OutstandingAmount = outstandingAmount
- Represents the remaining unpaid amount for the month.
- Calculated as:
```yaml
DueAmount - PaidAmount
```

#### PaymentStatus = monthlyStatus
- Represents the calculated payment status for the month.
- Determined based on the comparison between PaidAmount and DueAmount:
  - Unpaid" → No payment made
  - "Partially" → Partial payment made
  - "Paid" → Full payment completed

---

✅ Why this is better
- Frontend is simpler — no need to sum payments or calculate status there.
- Database stays raw — we only store transactions, never summary or calculated status.
- Handles any number of payments per month, automatically.
- Totals (totalDue, totalPaid, totalOutstanding) are always accurate.
