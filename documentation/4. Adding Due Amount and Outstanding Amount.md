# Adding Due Amount and Outstanding Amount

## Overview

This implementation introduces **Due Amount** and **Outstanding Amount**
calculations for student payments.

The system retrieves payment records for a student and calculates:

-   **Due Amount** -- The required monthly fee for the student's class.
-   **Outstanding Amount** -- The remaining amount that has not yet been
    paid.
-   **Total Due** -- The sum of all monthly fees for the queried
    records.
-   **Total Outstanding** -- The total remaining unpaid balance.

The monthly fee is retrieved from the **Classes entity**, which
represents the tuition class a student belongs to.

------------------------------------------------------------------------

# Payment Calculation Logic

## Concept

Each student belongs to a **class**, and each class defines a
**MonthlyFee**.

When retrieving payment data:

1.  The system retrieves the **student record**.
2.  The student's **class information** is also retrieved.
3.  The class provides the **MonthlyFee**, which becomes the **Due
    Amount** for each month.
4.  The system compares the **PaidAmount** stored in the Payments table
    with the **MonthlyFee**.

------------------------------------------------------------------------

## Core Calculation Logic

For every payment record:

**Due Amount** - Equal to the **MonthlyFee** defined in the student's
class.

**Outstanding Amount** - Calculated by subtracting the paid amount from
the due amount.

Formula:

OutstandingAmount = DueAmount − PaidAmount

Example:

```yaml
  Monthly Fee   Paid Amount   Outstanding
  ------------- ------------- -------------
  100           0             100
  100           50            50
  100           100           0
```

------------------------------------------------------------------------

## Total Calculation

The system also calculates overall totals while processing each payment.

During iteration:

TotalDue += DueAmount\
TotalOutstanding += OutstandingAmount

Example:

```yaml
  Month   Due   Paid   Outstanding
  ------- ----- ------ -------------
  Jan     100   100    0
  Feb     100   50     50
  Mar     100   0      100
```

Result:

TotalDue = 300
TotalOutstanding = 150

------------------------------------------------------------------------

# Data Flow

1.  API receives request containing student IC number.
2.  Query handler calls the repository to retrieve the student record.
3.  Repository returns:
    -   Student details
    -   Payment records
    -   Class information
4.  Query handler retrieves the **MonthlyFee** from the class.
5.  For each payment record:
    -   Due Amount is determined.
    -   Outstanding Amount is calculated.
    -   Totals are accumulated.
6.  Data is mapped into **PaymentDto** objects.
7.  All results are packaged into **StudentPaymentResponseDto**.
8.  API returns the final response.

Pseudo Code
```yaml
MonthlyDue = 100;  // We take value 100 from Classes Entity
totalDue = 0;
totaloutstanding = 0;

Start

DueAmount = MonthlyDue;
OutStandingAmount = DueAmount - PaidAmount(Payment Entity)

totalDue += DueAmount;
totalOutstanding += OutStandingAmount;

Pass All The Value in PaymentDto

End

Pass The Final Response To Display In API

```

# Entity Changes

## Payments Entity

Two additional details are added to support payment tracking:

-   **PaidAmount**
    -   Stores the actual amount paid by the student.
-   **PaymentStatus**
    -   Enum used to represent the payment state.

Possible statuses:

-   Paid
-   Unpaid
-   Partially

### File: Payments.cs

``` csharp
namespace mktsystem.domain.Entities;

public class Payments
{
    public int Id { get; set; }
    public Students? Student { get; set; }
    public int StudentId { get; set; }
    public int Month { get; set; }
    public int Year { get; set; }
    
    public decimal PaidAmount { get; set; } // store what was actually paid
    public PaymentStatus PaymentStatus { get; set; }
    public DateTime CreatedAt { get; set; }
    public DateTime? UpdatedAt { get; set; }
}

public enum PaymentStatus
{
    Paid,
    Unpaid,
    Partially   // If user paid partially
}
```

# Data Seeding

Since payment calculations depend on **Class MonthlyFee**, initial class
data must exist.

### File: IClassSeeder.cs

``` csharp
}
```

### File: ClassSeeder.cs

``` csharp
public class ClassSeeder(MktSystemDbContext dbContext) : IClassSeeder
{
    public async Task Seed()
    {
        if (!dbContext.Classes.Any())
        {
            var classes = new List<Classes>
            {
                new Classes { Name = "Class A", MonthlyFee = 100 },
                new Classes { Name = "Class B", MonthlyFee = 120 },
                new Classes { Name = "Class C", MonthlyFee = 150 }
            };

            await dbContext.Classes.AddRangeAsync(classes);
            await dbContext.SaveChangesAsync();
        }
    }
}
```

# Query Handler Logic

The query handler performs the main calculation process.

Responsibilities:

1.  Retrieve student data
2.  Retrieve class monthly fee
3.  Loop through payment records
4.  Calculate due and outstanding amounts
5.  Accumulate totals
6.  Convert results into DTO objects
7.  Return structured response to the API

Full Code:
```csharp
namespace mktsystem.application.StudentPayment;

public class StudentPaymentQueryHandler(
    ILogger<StudentPaymentQueryHandler> logger,
    IStudentRepository studentRepository
) : IRequestHandler<StudentPaymentQuery, StudentPaymentResponseDto>
{
    public async Task<StudentPaymentResponseDto> Handle(StudentPaymentQuery request, CancellationToken cancellationToken)
    {
        logger.LogInformation("Retrieving payments for IC: {IcNumber}", request.IcNumber);

        // Get student + payments from repository
        var student = await studentRepository.GetStudentWithPaymentsByIc(request.IcNumber, cancellationToken);

        if (student == null)
        {
            logger.LogWarning("Student not found: {IcNumber}", request.IcNumber);
            return null; // Controller can return 404
        }

        // Get monthly fee from the class
        var monthlyFee = student.Class.MonthlyFee;

        decimal totalDue = 0;
        decimal totalOutstanding = 0;

        // Build DTO list manually with calculated fields
        var paymentDtos = new List<PaymentDto>();
        foreach (var payment in student.Payments)
        {
            var dueAmount = monthlyFee;
            var outstandingAmount = dueAmount - payment.PaidAmount;

            totalDue += dueAmount;
            totalOutstanding += outstandingAmount;

            paymentDtos.Add(new PaymentDto
            {
                Month = payment.Month.ToString(),
                Year = payment.Year,
                PaymentStatus = payment.PaymentStatus.ToString(),
                PaidAmount = payment.PaidAmount,
                DueAmount = dueAmount,
                OutstandingAmount = outstandingAmount
            });
        }

        // Build final response
        var response = new StudentPaymentResponseDto
        {
            Name = student.Name,
            IcNumber = student.IcNumber,
            Payments = paymentDtos,
            TotalDue = totalDue,
            TotalOutstanding = totalOutstanding
        };

        return response;
    }
}
```
