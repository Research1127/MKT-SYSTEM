# Retrieving Student Payment Data

This API retrieves a student’s payment history based on their IC number. It is built using **ASP.NET Core**, **Entity Framework Core**, **MediatR**, and **AutoMapper**.

---

## 1. Database & Repository Layer (Infrastructure)

In `Infrastructure Layer`

### File: Repositories/StudentRepository.cs

```csharp
using Microsoft.EntityFrameworkCore;
using mktsystem.domain.Entities;
using mktsystem.domain.Repositories;
using mktsystem.infrastructure.Persistence;

namespace mktsystem.infrastructure.Repository;

public class StudentRepository(MktSystemDbContext dbContext) : IStudentRepository
{
    public async Task<Students?> GetStudentWithPaymentsByIc(string icNumber, CancellationToken cancellationToken)
    {
        // Retrieves a student and their payments from the database using IC number
        return await dbContext.Students
            .Include(s => s.Payments) // Include related payments
            .FirstOrDefaultAsync(s => s.IcNumber == icNumber, cancellationToken);
    }
}
```

**Explanation:**

- **Repository Pattern**: Provides a clean interface to access data without exposing EF Core directly.
- `.Include(s => s.Payments)` ensures we fetch related payment records together.
- `CancellationToken` allows the query to be cancelled if the request is aborted.

In `Domain Layer`

### File: Repositories/IStudentRepository.cs

```csharp
using mktsystem.domain.Entities;

namespace mktsystem.domain.Repositories;

public interface IStudentRepository
{
    Task<Students?> GetStudentWithPaymentsByIc(string icNumber, CancellationToken cancellationToken);
}
```

**Explanation:**

- Defines the contract for student data access.
- Any repository implementation must provide this method.
- Keeps the application layer independent from EF Core.

## 3. AutoMapper Profile

In `Application Layer`

### File: Dtos/PaymentProfile.cs

```csharp
using AutoMapper;
using mktsystem.application.Dtos;
using mktsystem.domain.Entities;

public class PaymentProfile : Profile
{
    public PaymentProfile()
    {
        CreateMap<Payments, PaymentDto>()
            .ForMember(dest => dest.Month, opt => opt.MapFrom(src =>
                new DateTime(src.Year, src.Month, 1).ToString("MMMM"))) // Convert month number to name
            .ForMember(dest => dest.PaymentStatus, opt => opt.MapFrom(src => src.PaymentStatus.ToString()));
    }
}
```

**Explanation:**

- Converts EF Core entity (`Payments`) to `PaymentDto`.
- Maps month number to month name (e.g., 1 → "January").
- Converts enum `PaymentStatus` to string for client readability.

## 4. MediatR Query

In `Application Layer`

### File: StudentPayment/StudentPaymentQuery.cs

```csharp
using MediatR;
using mktsystem.application.Dtos;

namespace mktsystem.application.StudentPayment;

public record StudentPaymentQuery(string IcNumber) : IRequest<StudentPaymentResponseDto>;
```

**Explanation:**

- Defines the **query** request.
- Contains only the IC number of the student.
- Used by **MediatR** to trigger the handler.

### File: StudentPayment/StudentPaymentQueryHandler.cs

```csharp
using AutoMapper;
using MediatR;
using Microsoft.Extensions.Logging;
using mktsystem.application.Dtos;
using mktsystem.domain.Repositories;

public class StudentPaymentQueryHandler(
    ILogger<StudentPaymentQueryHandler> logger,
    IMapper mapper,
    IStudentRepository studentRepository
) : IRequestHandler<StudentPaymentQuery, StudentPaymentResponseDto>
{
    public async Task<StudentPaymentResponseDto> Handle(StudentPaymentQuery request, CancellationToken cancellationToken)
    {
        logger.LogInformation("Retrieving payments for IC: {IcNumber}", request.IcNumber);

        var student = await studentRepository.GetStudentWithPaymentsByIc(request.IcNumber, cancellationToken);

        if (student == null)
        {
            logger.LogWarning("Student not found: {IcNumber}", request.IcNumber);
            return null; // Controller can handle NotFound
        }

        var response = new StudentPaymentResponseDto
        {
            Name = student.Name,
            IcNumber = student.IcNumber,
            Payments = mapper.Map<List<PaymentDto>>(student.Payments)
        };

        return response;
    }
}
```

**Explanation:**

- Handles the query request.
- Fetches student from repository.
- Maps payments to `PaymentDto` using AutoMapper.
- Logs actions for monitoring and debugging.
- Returns `null` if student not found (controller decides response).

## 5. Controller 

In `Api Layer`

### File: PaymentsController.cs

```csharp
using MediatR;
using Microsoft.AspNetCore.Mvc;
using mktsystem.application.StudentPayment;
using mktsystem.application.Dtos;

namespace mktsystem.Controllers;

[ApiController]
[Route("api/payments")]
public class PaymentsController(IMediator mediator) : ControllerBase
{
    [HttpGet("{icNumber}")]
    public async Task<IActionResult> GetStudentPaymentsByIc(string icNumber)
    {
        var result = await mediator.Send(new StudentPaymentQuery(icNumber));

        if (result == null)
            return NotFound(new { Message = "Student not found" });

        return Ok(result);
    }
}
```

**Explanation:**

- Receives HTTP GET requests at `/api/payments/{icNumber}`.
- Uses MediatR to send the query to the handler.
- Returns `404 Not Found` if student does not exist.
- Returns `200 OK` with student payment details if found.
