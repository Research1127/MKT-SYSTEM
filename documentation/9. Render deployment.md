# 📘 Mktsystem Deployment on Render with Supabase

## Overview

This document explains how to deploy the **mktsystem** backend on **Render** using **Docker**, with database connectivity via **Supabase session puller**, including environment variables, ports, and optional CORS setup.

## Repository Setup

- **Git Repository**: `MKT-SYSTEM`  
- **Solution file**: `mktsystem.sln`  
- **Dockerfile Path**: `mktsystem/Dockerfile`  


> Notes: Dockerfile Path tells Render where to look for the Dockerfile.

## File: Dockerfile

- Dockerfile should already exist in `mktsystem/`.  
- Ports exposed: **10000** (or adjust if needed).

```dockerfile
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS base
WORKDIR /app

# ✅ Use port 10000 for Render
EXPOSE 10000
ENV ASPNETCORE_URLS=http://+:10000

FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src

# Copy solution file
COPY mktsystem.sln .

# Copy project files
COPY mktsystem/mktsystem.csproj mktsystem/
COPY mktsystem.application/mktsystem.application.csproj mktsystem.application/
COPY mktsystem.domain/mktsystem.domain.csproj mktsystem.domain/
COPY mktsystem.infrastructure/mktsystem.infrastructure.csproj mktsystem.infrastructure/

# Restore dependencies
RUN dotnet restore mktsystem/mktsystem.csproj

# Copy the rest of the code
COPY . .

# Build and publish
WORKDIR /src/mktsystem
RUN dotnet publish -c Release -o /app/publish

FROM base AS final
WORKDIR /app
COPY --from=build /app/publish .

ENTRYPOINT ["dotnet", "mktsystem.dll"]
```

- Build and publish are handled by the Dockerfile using the multi-stage approach.

## Environment Variables

In Render → Web Service → **Settings → Environment**  

````yaml
| Name                                   | Value                                                                                                                                  |
|----------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------|
| `ASPNETCORE_ENVIRONMENT`               | Production                                                                                                                             |
| `CONNECTIONSTRINGS__DEFAULTCONNECTION` | Host=db-pooler.xxx.supabase.co;Port=5432;Database=postgres;Username=postgres;Password=xxxx;SSLMode=Require;TrustServerCertificate=true |

````
> ⚠️ Make sure the connection used Direct Connection String and choose Session Pooler in Supabase Connection Settings

> ⚠️ Replace `xxxx` with your actual Supabase password.  

> ⚠️ Ensure SSLMode and TrustServerCertificate are included for secure connection.

> ⚠️ Ensure `ASPNETCORE_ENVIRONMENT=Production` required by Render

## Supabase Configuration

1. Go to Supabase → Main Dashboard → **Connect**  
2. Use the **Direct Connection String** → **Session Pooler** for Render connection  
3. Ensure your database allows external IPv4 connections (required by Render)

## CORS Setup (Optional)

If frontend is on a different domain (e.g., React), configure CORS in `Program.cs`:

```csharp
builder.Services.AddCors(options =>
{
    options.AddDefaultPolicy(policy =>
    {
        policy.WithOrigins("https://yourfrontend.com")
              .AllowAnyHeader()
              .AllowAnyMethod();
    });
});

app.UseCors();
```

## Health Check (Optional)

Add a simple health endpoint to verify the service is running:

```csharp
app.MapGet("/health", () => "API is running");
```

Render can use this to monitor service status.

## Render Deployment Steps

1. **Create Web Service** on Render → choose Docker  
2. **Select Repository** → MKT-SYSTEM
3. **Set DockerFile Path Directory** → `mktsystem/Dockerfile`  
4. **Set Environment Variables** (see Environment Variables section)  
5. **Deploy** → Render builds the Docker image and starts the service  
6. Check **Logs** for successful startup:

```text
Available at your primary URL https://mkt-system-f225.onrender.com
```
