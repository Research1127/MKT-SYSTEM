# 📘 Use Fee Rules for Monthly Fee Calculation

This feature calculates a student's **monthly fee dynamically** based on
their **family income**, using predefined ranges stored in the `Fees`
table.

------------------------------------------------------------------------

# 🧠 Concept Overview

Instead of storing a fixed fee in the `Students` or `Classes` table, we:

-   Store **income ranges** in the `Fees` table
-   Match a student's `FamilyIncome` against those ranges
-   Return the corresponding `FeeAmount`

------------------------------------------------------------------------

# 🗃️ Fees Table Structure

```yaml
Column       |   Description
-------------|----------------------------------------
MinIncome    |   Minimum income for the range
MaxIncome    |   Maximum income (nullable for no limit)
FeeAmount    |   Monthly fee to apply
```

------------------------------------------------------------------------

# 🏗️ Implementation

------------------------------------------------------------------------

# 1️⃣ Add New Method in Repository

## 📁 File: `StudentRepository.cs`

``` csharp
public async Task<decimal> GetFeeAmount(decimal familyIncome, CancellationToken cancellationToken)
{
    var fee = await dbContext.Fees
        .FirstOrDefaultAsync(f => familyIncome >= f.MinIncome &&
                                  (f.MaxIncome == null || familyIncome <= f.MaxIncome),
            cancellationToken);

    return fee?.FeeAmount ?? 0;
}
```

------------------------------------------------------------------------

## 🔍 Detailed Explanation (IMPORTANT)

This method is responsible for:

👉 **Finding the correct fee based on income range**

------------------------------------------------------------------------

### 🔹 Step 1: Query the Fees table

``` csharp
var fee = await dbContext.Fees.FirstOrDefaultAsync(...)
```

-   Queries the database
-   Returns the **first matching fee record**
-   Result type: `Fees?` (nullable)

------------------------------------------------------------------------

### 🔹 Step 2: Apply income range logic

``` csharp
f => familyIncome >= f.MinIncome &&
     (f.MaxIncome == null || familyIncome <= f.MaxIncome)
```

------------------------------------------------------------------------

#### ✅ Lower bound check

``` csharp
familyIncome >= f.MinIncome
```

------------------------------------------------------------------------

#### ✅ Upper bound check

``` csharp
f.MaxIncome == null || familyIncome <= f.MaxIncome
```

------------------------------------------------------------------------

### 🔹 Step 3: Return FeeAmount safely

``` csharp
return fee?.FeeAmount ?? 0;
```

🔹 Part A: fee?.FeeAmount

👉 Means:
- If fee exists → return FeeAmount
- If fee is null → return null (no crash)

⸻

🔹 Part B: ?? 0

👉 Means:
- If result is null → replace with 0

------------------------------------------------------------------------

### 🔥 Combined meaning

```yaml
If fee exists → return FeeAmount
If no match → return 0
```
