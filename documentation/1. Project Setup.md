## Create Entities/Students.cs file in Domain Module
```csharp
namespace mktsystem.domain.Entities;

public class Students
{
    public int Id { get; set; }
    public string Name { get; set; }
    public string IC_Number { get; set; }
    public string Class { get; set; }
    public int Year { get; set; }
    public List<Payments> Payments { get; set; } = new List<Payments>();
    public DateTime CreatedAt { get; set; }
    public DateTime? UpdatedAt { get; set; }
}
```

Explanation:
- `Id` → Primary key of the Students table. EF Core automatically uses Id as the PK.
- `Name, IC_Number, Class, Year` → Columns storing student info.
- `Payments` → Navigation property for one-to-many relationship (one student can have multiple payments). Initialized with an empty list to avoid null errors.
- `CreatedAt and UpdatedAt` → Track when a student record is created or updated.

## Create Entities/Payments.cs file in Domain Module

```csharp
namespace mktsystem.domain.Entities;

public class Payments
{
    public int Id { get; set; }
    public Students? Student { get; set; }
    public int StudentId { get; set; }
    public int Month { get; set; }
    public int Year { get; set; }
    public PaymentStatus PaymentStatus { get; set; }
    public DateTime CreatedAt { get; set; }
    public DateTime? UpdatedAt { get; set; }
}

public enum PaymentStatus
{
    Paid,
    Unpaid,
}
```

Explanation:
- `Id` → Primary key for Payments table.
- `StudentId` → Foreign key referencing the Students table.
- `Students?` → Navigation property to access the related student object in code. 
It nullable because we can create payment without student object like name or ic. Just use the student id to relate
- `Month and Year` → Which month/year this payment is for.
- `PaymentStatus` → Enum to track if the payment is paid or unpaid.
- `CreatedAt` and UpdatedAt → Track creation and update times.

## Create `Persistence/MktSystemDbContext.cs` in `Infrastructure Module`

```csharp
using Microsoft.EntityFrameworkCore;
using mktsystem.domain.Entities;

namespace mktsystem.infrastructure.Persistence;

public class MktSystemDbContext : DbContext
{
    public MktSystemDbContext(DbContextOptions<MktSystemDbContext> options) : base(options)
    {

}
    
    public DbSet<Students> Students  { get; set; }
    public DbSet<Payments> Payments  { get; set; }

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        base.OnModelCreating(modelBuilder);

        modelBuilder.Entity<Payments>()
            .HasOne(c => c.Student)
            .WithMany(p  => p.Payments)
            .HasForeignKey(k => k.StudentId)
            .OnDelete(DeleteBehavior.Cascade);
    }

}

```

Explanation:
- DbSet<Students> Students → Represents the Students table in the database. Allows querying all students.
- DbSet<Payments> Payments → Represents the Payments table.
- OnModelCreating → Configures relationships between entities:
  - HasOne(c => c.Student) → Each Payment is related to one Student.
  - WithMany(p => p.Payments) → Each Student can have many Payments.
  - HasForeignKey(k => k.StudentId) → Payment table column storing the FK.
  - OnDelete(DeleteBehavior.Cascade) → If a student is deleted, all their payments are automatically deleted.

💡 Note: EF Core uses the navigation property (Students in Payments) to understand relationships. Without it, you can’t do things like:

```csharp
var student = dbContext.Students.Include(s => s.Payments).First();
```

## Create `Extensions/ServiceCollectionExtensions.cs` in `Infrastructure layer` and register the `MktSystemDbContext` there and register the connection
```csharp
namespace mktsystem.infrastructure.Extensions;

public static class ServiceCollectionExtensions
{
    public static void AddInfrastructure(this IServiceCollection services, IConfiguration configuration)
    {
        var connectionString = configuration.GetConnectionString("DefaultConnection");
        
        services.AddDbContext<MktSystemDbContext>(options => options.UseNpgsql(connectionString));
    }
}
```

## Register the extensions in `program.cs` and add automatic migrate
```csharp

// Register The Infrastructure Extensions
builder.Services.AddInfrastructure(builder.Configuration);

// Other code here
// Run migrations automatically
using (var scope = app.Services.CreateScope())
{
    var dbContext = scope.ServiceProvider.GetRequiredService<MktSystemDbContext>();
    dbContext.Database.Migrate();
}
```

Do migration
```shell
dotnet ef migrations add InitialCreate -p ./mktsystem.infrastructure -s ./mktsystem
dotnet ef database update -p ./mktsystem.infrastructure -s ./mktsystem
```
